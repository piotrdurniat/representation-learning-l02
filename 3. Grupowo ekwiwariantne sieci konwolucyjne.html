3. Grupowo ekwiwariantne sieci konwolucyjne¶

Ostatnią architekturą, którą dzisiaj sprawdzimy są tzw. Group Equivariant Neural Networks, czyli sieci neuronowe (w naszym przypadku konwolucyjne), które opisują funkcje ekwiwariantne względem określonej grupy symetrii. Po raz pierwszy koncepcja takich sieci została opisana w 2016 r. przez Cohena i Wellinga w pracy Group Equivariant Convolutional Networks. W pracy formalizują zastosowanie grup symetrii do modelowania struktury danych – dokładniej mówiąc rozważają dwie grupy: p4 (obroty o 90 stopni) oraz p4m (obroty o 90 stopni i odbicia).

Implementacja grupowo ekwiwariantnych sieci jest zadaniem nietrywialnym i czasochłonnym, stąd w tym zeszycie wykorzystamy bibliotekę e2cnn, która posiada zaimplementowane warstwy grupowo ekwiwariantne.

3.1. Architektura¶

Założeniem modelu będzie, aby koder był w stanie stworzyć niezmienniczą (ang. invariant) reprezentację podstawowych i obróconych obrazów wejściowych. Moduł dekodera nie będziemy modyfikować i użyjemy klasy CNNDecoder z poprzedniego zeszytu.

from e2cnn import gspaces
from e2cnn import nn as enn
import torch.nn as nn


class GCNNEncoder(torch.nn.Module):

    def __init__(self, latent_dim: int, N: int):
        super().__init__()

        # group of symmetries (2D rotations)
        self.symm_group = gspaces.Rot2dOnR2(N=N)

        self.input_field_type = enn.FieldType(
            self.symm_group, [self.symm_group.trivial_repr]
        )
        self.hidden_field_type_1 = enn.FieldType(
            self.symm_group, 8 * [self.symm_group.trivial_repr]
        )
        self.hidden_field_type_2 = enn.FieldType(
            self.symm_group, 16 * [self.symm_group.trivial_repr]
        )
        self.outpu_field_type = enn.FieldType(
            self.symm_group, 32 * [self.symm_group.trivial_repr]
        )

        self.layers = enn.SequentialModule(
            enn.R2Conv(
                self.input_field_type,
                self.hidden_field_type_1,
                kernel_size=3,
                stride=2,
                padding=1,
            ),
            enn.ReLU(self.hidden_field_type_1, inplace=True),
            enn.R2Conv(
                self.hidden_field_type_1,
                self.hidden_field_type_2,
                kernel_size=3,
                stride=2,
                padding=1,
            ),
            enn.ReLU(self.hidden_field_type_2, inplace=True),
            enn.R2Conv(
                self.hidden_field_type_2,
                self.outpu_field_type,
                kernel_size=3,
                stride=2,
                padding=1,
            ),
            enn.PointwiseMaxPoolAntialiased(
                in_type=self.outpu_field_type, kernel_size=3
            ),
            enn.GroupPooling(self.outpu_field_type),
        )

        self.mlp = torch.nn.Sequential(
            nn.Flatten(),
            nn.Linear(32, 128),
            nn.ReLU(inplace=True),
            nn.Linear(128, latent_dim),
        )

    def forward(self, img: torch.Tensor) -> torch.Tensor:
        img_geo = enn.GeometricTensor(img, self.input_field_type)
        x = self.layers(img_geo)
        x = x.tensor
        x = self.mlp(x)
        return x

import torch
import torchvision.transforms.functional as TF


def rotate(img: torch.Tensor, angle: float) -> torch.Tensor:
    pil_image = TF.to_pil_image(img, mode="F")
    rotated_pil_image = TF.rotate(pil_image, angle)
    rotated_tensor = TF.to_tensor(rotated_pil_image)
    rotated_tensor = rotated_tensor.squeeze(0)
    return rotated_tensor

Zadanie 3.4 (1 pkt)¶

Czy autokoder G-CNN jest odporny na obroty obiektów? Z czego może to wynikać?

Student's answer Score: 0.5 / 1.0 (Top)

Autokoder G-CNN jest częściowo odporny na obroty obiektów, dla małych kątów obrotu reprezentacja obiektu zmienia się w niewielkim stopniu. Wynika to z faktu, że sieć wykorzystuje grupę obrotów jako grupę symetrii. W przypadku dużych kątów obrotu, sieć nie jest w stanie zachować ekwiwariancji.
